# 用户故事 · 运营商 / 渠道商 / 资金方后台

> 与 [PRD.md](./PRD.md)、[acceptance-criteria.md](./acceptance-criteria.md) 配套。

## 运营商（经营主体）

- 作为运营商，我希望只管理**自有**换电柜与电池（含租赁设备），以便权责清晰。
- 作为运营商，我希望只看到**自有设备产生**的套餐/换电订单与流水，以便对账准确。
- 作为运营商，我希望管理自有站点，并在「合作伙伴分润」中为站点指定合作方（数据来自员工档案）。
- 作为运营商，我希望在**「员工」**中维护运营员工（配置权限）与**合作伙伴**（分润比例、关联站点），以便分工与对账。
- 作为运营商，我希望在**「合作伙伴分润」**查看关联站点的订单明细（用户 ID/手机完整），并与汇总、结算批次勾稽。
- 作为运营商，我希望管理租赁协议与月租金，在收益不足时人工处理缺口。
- 作为运营商，我希望管理个人套餐定价、渠道批发与人天额度销售，并查看跨运营商往来账与平台服务费。

## 渠道商（经营主体）

- 作为渠道商，我希望向签约运营商采购人天额度，并登记、分配团队骑手额度。
- 作为渠道商，我希望查看额度消耗明细与低余额预警，以便及时续费。

## 资金方（经营主体）

- 作为资金方，我希望管理出租协议、绑定设备及承租方还款进度。
- 作为资金方，我希望在**「租金收缴」**按账期跟踪应收/已收与跟进状态。

## 运营员工（员工登录）

- 作为运营员工，我希望用独立账号登录后台，**只看到被授权的菜单**（如设备、订单、用户），以便最小权限协作。
- 作为运营员工，我希望看到的数据范围与所属经营主体一致（`device_owner_id`），不能查看其他主体数据。

## 合作伙伴（合作伙伴登录）

- 作为合作伙伴，我希望用独立账号登录，在**「个人账户」**查看分润余额、绑定银行卡并**申请提现**。
- 作为合作伙伴，我希望在**「合作伙伴分润」**仅查看**与我关联站点**的套餐/换电订单明细，以便与主体对账。
- 作为合作伙伴，我希望**不能**修改分润设置或管理其他合作方，以免越权。

## 品牌方

- 作为品牌方，我不使用本后台管理运营设备（仅提供平台与用户端，非本期）。
