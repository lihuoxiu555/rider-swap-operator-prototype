# 验收标准 · 四角色后台 + 骑手端（原型）

## 通用

- `python3 main.py` 端口 `8766` 可预览后台、骑手端、**产品文档浏览器**（`/prototype/docs/index.html`，渲染 `docs/*.md`）
- 侧栏展示当前登录主体名称与角色类型
- 可切换演示角色：**平台管理员** / **运营商** / **渠道商** / **资金方**；导航随角色变化
- 侧栏「切换登录身份」含：经营主体、员工登录、合作伙伴登录（运营商合作伙伴）
- 无全局筛选条；各页面独立筛选区；**无「重置筛选」按钮**
- 模块「说明」弹窗可用；无代理层级/下级分润表述
- **产品文档**：`/prototype/docs/index.html` 可浏览 `docs/*.md`（含 [设计评审报告.md](./设计评审报告.md)、[额度池PRD差异清单.md](./额度池PRD差异清单.md)；侧栏切换、Markdown 渲染、**Mermaid 流程图渲染为图形**）

## 平台管理员

| 模块 | 验收项 | 预期 |
|------|--------|------|
| 总览 | 业务汇总 KPI | 运营商数、渠道商数、用户数、站点数、设备数、订单数、业务流水、平台营收 |
| 总览 | 运营商账户汇总表 | 各运营商保证金/信用额度/跨网状态；可进详情抽屉 |
| 用户管理 | 服务运营商 | 列展示当前服务运营商（个人=套餐售卖方；渠道=额度售卖方） |
| 用户管理 | 套餐与人天 | 套餐状态+生效周期；渠道成员展示渠道商、人天额度、今日资格 |
| 用户管理 | 持有电池 | 服务中且已换电：展示 SN/SOC/归属；**已冻结**→未持有（须先还电）；**中途完结**→未持有（已还电）；**待首换开通**→未持有 |
| 用户管理 | 冻结校验 | 骑手端「申请冻结」：持有电池时提示「请先归还电池后再申请冻结」；还电后可提交 |
| 订单管理 | 三 Tab | 套餐购买 / 换电 / 渠道商批发；全平台数据可筛选 |
| 订单管理 | 换电三元组 | 换电 Tab 含 user/柜机/电池运营商与跨网服务费 |
| 设备管理 | 三 Tab | 设备台账 / 待绑定 / 批量导入 |
| 设备管理 | 批量导入 | 粘贴 SN 列表导入待绑定队列（演示） |
| 设备管理 | 绑定 | 待绑定设备可绑定至在营运营商 |
| 渠道商管理 | 列表 | 只读；账号、创建/维护方（运营商）、额度池、人天、人员规模 |
| 渠道商管理 | 人员明细 | 只读；登记骑手、**团队**、额度状态 |
| 渠道商管理 | 不可新增 | 平台侧无「新增渠道商」入口 |
| 流水管理 | 三 Tab | 用户支付 / 运营商之间 / 平台提成 |
| 流水管理 | 平台 1% | 用户支付与平台提成 Tab 高亮 1% 金额 |
| 平台账户 | 商户与营收 | 微信/支付宝商户、余额、本月 C 端+B 端提成构成 |
| 运营商管理 | 列表与筛选 | 名称/ID/城市/状态筛选；新增/编辑（演示 Mock） |
| 运营商管理 | 详情抽屉 | 基础信息、清分账户（保证金+信用额度）、进件摘要、经营汇总 |
| 跨网统一定价 | 编辑发布 | 柜机/电池服务费可改（本地 Mock）；**人天标准日值**可改（默认 ¥8.5/人天）；运营商只读展示 |
| 登录 | 切换身份 | 登录下拉首项「平台管理员 · 智格超能」 |

## 运营商

| 模块 | 验收项 | 预期 |
|------|--------|------|
| 总览 | KPI | 柜机在线、待结转/可打款、合作伙伴待付、活跃用户；**不含**人天池 KPI |
| 总览 | 跨运营商 | 独立卡片展示待日清**应收/应付**汇总（Mock） |
| 总览 | 平台服务费 | 本月计提、待代扣金额（Mock） |
| 定价 | 个人套餐 | 城市+站点+SKU 零售价；含包月/7天/**1天/单次**；可维护（演示） |
| 定价 | 渠道兜底 SKU | **强制**展示 1天+单次 各至少一款 `channelFallback` 标记；缺失时告警 |
| 定价 | 人天批发 | 展示平台标准价（只读）与「我的批发价」；默认批发价=平台标准价，运营商可改（Mock：`OP-Q-02` 默认 ¥9.0） |
| 渠道销售 | 签约渠道 | 运营商可「+ 新增渠道商」；维护主体、批发价、合同有效期；**不需配置可用站点**（由渠道额度使用规则维护）；可编辑；**新增时自动创建待配置额度池 + 默认团队** |
| 渠道销售 | 已售额度池 | 「额度调整」含充值/赠送/退款/修正；**过期恢复**（池过期后 30 天内，如 `QP-2603` 演示） |
| 渠道销售 | 批发订单 | `PO-202601-088` 等；渠道、人天数、金额、付款状态 |
| 渠道销售 | 卖方额度池 | 运营商视角：**一渠道一行**；多次采购不新建池，见**额度变动记录**；可「额度调整」（充值/赠送/退款/修正/过期恢复） |
| 站点/设备/订单/流水 | 数据隔离 | 仅 `OP-SX` 名下设备与订单 |
| 我的设备 | 平台绑定前置 | 页顶提示：设备须先由平台管理员绑定 |
| 站点 | 换电范围设置 | 两个开关：允许跨网换电、允许跨站点换电；可即时切换 |
| 站点 | 关跨网 | 关闭后双向：本运营商用户不可出站；外来用户不可进站 |
| 站点 | 关跨站 | 关闭后用户仅可在主绑定站点换电（同运营商其他站拦截） |
| 往来账 | 平台代收代付 | 不展示对手方运营商；仅「平台代收」「平台代付」及金额 |
| 往来账 | 统价高亮 | 柜机 ¥0.5/次、电池 ¥0.1/次（暂定，平台管控） |
| 往来账 | 日清 | 每日 23:59:59 日账单；周/月汇总表 |
| 往来账 | 保证金/信用额度 | KPI：保证金余额优先展示；保证金=0 时展示信用额度占用；额度用光停跨站（Mock OP-BJ） |
| 往来账 | 案例 #3 | 个人用户跨站：U 平台代付 ¥0.5+¥0.1；渠道跨网规则与个人一致 |
| 平台服务费 | 计提 | **已确认**按「确认消耗」：**平台标准人天价** × 1% 向**额度售卖方 U** 计提（与合同批发价无关；Mock `PF-004` 批发 ¥9.0 仍按 ¥8.5 计提）；**渠道消耗不向运营商二次打款**（B2B 采购已结清） |
| 平台服务费 | 代扣 | 保证金代扣（优先）/ 信用额度代扣（保证金=0）演示 |
| 员工/合作伙伴 | 同前 | 运营员工权限；合作伙伴分润 ★ |
| 合作伙伴分润 | 默认比例 | 无合伙人站点展示「平台 1% + 运营商 99%」 |
| 合作伙伴分润 | 合伙人 30% | 站点 A 展示「平台 1% + 合伙人 30% + 运营商 69%」 |
| 合作伙伴分润 | 尾差 | 订单明细：平台、合伙人各 3 位小数；运营商吸收余额（例 B=9.96 R=30%） |
| 合作伙伴分润 | 范围 | 仅个人用户订单；渠道人天用户不出现在分润明细 |
| 合作伙伴分润 | 汇总勾稽 | 分润汇总「合伙人计提」= 订单明细合伙人份额之和 |
| 合作伙伴分润 | 生效时间 | 比例/站点变更进入「待生效规则」；生效时间为次日 0:00；Mock：P01 待 30%→25% |
| 租赁 | 承租 | 协议与设备、月租金、收益不足缺口演示 |

| 额度池 | 列表 | `QP-2601` owner=`CH-SF`；余额 18.5% 预警 |

| 额度池 | 骑手日消耗 | 每骑手每日 1 条；字段：换电次数、持有电池数、确认原因 |
| 额度池 | 换电同步 | 每次渠道骑手换电实时同步至渠道商 |
| 额度池 | 零售价 | **只读**；含 1天/单次 兜底价；权益不可用时引导在线支付 |
| 额度池 | 签约运营商 | 可签约多家运营商；演示 闪送驿站 + 陆家嘴联营；**2 个额度池**（一运营商一池） |
| 额度池 | 合同配置 | 额度使用规则 Tab 展示四项合同配置（扣天/激活/退款/结算） |
| 额度池 | B2B 退款 | 渠道端展示「不支持在线退款」说明；无自助退池入口 |
| 额度池 | 额度使用规则 | 选团队 → 自动带出消耗池；配置**团队周期上限**；无站点/权益类型字段 |
| 员工 | 额度运营 | 如「张渠道运营」：额度池相关权限 |
