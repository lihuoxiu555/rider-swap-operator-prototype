# 个人套餐定价 · 渠道兜底 · 自动退款

> 运营商 C 端套餐 SKU、站点定价、渠道无预占自费、平台 1% 与中途退款规则。  
> 关联：[PRD.md](./PRD.md) §5.2 / §5.4 · [天数池.md](./天数池.md) · [合作模式与分账.md](./合作模式与分账.md) · [合伙人站点分佣.md](./合伙人站点分佣.md)

## 1. 套餐 SKU（四类）

| SKU | 类型码 | 有效期 | 换电 | 收入确认 |
|-----|--------|--------|------|----------|
| **包月** | `monthly` | 购买日起 D 自然日（如 30） | 期内不限次（限次另配） | 按日摊销 `P/D` |
| **7 天** | `weekly` | 7 自然日 | 同上 | 按日摊销 `P/7` |
| **1 天** | `daily` | **购买后 24h 内有效** | 同上 | 按日或首换当笔（与 SKU 配置一致） |
| **单次** | `single` | **购买后 24h 内有效** | 1 次换电即完结 | 换电成功当笔确认 |

> **已确认（Q3）**：1 天与单次均为 **购买时刻起 24 小时内有效**，非自然日 0:00 切日。

## 2. 站点定价

### 2.1 定价维度

```
运营商 × 城市 × 站点(可空) × SKU → 零售价
```

**继承优先级**：站点专属价 > 城市默认价 > 运营商全局默认价。

### 2.2 是否必须分站点不同价

| 场景 | 建议 |
|------|------|
| 同城无合伙人差异 | 城市级统一定价即可 |
| 站点绑定合伙人 / 场地成本不同 | 允许站点覆盖价 |
| 跨网换电 | 价格锁定在**购买套餐单**，不因换电站变化 |

## 3. 支付路径

### 3.1 个人用户

选择站点可用 SKU → **在线支付** → 支付成功 → 平台 **C 端 1%** 分账 → 生成 `PackageOrder`。

### 3.2 渠道成员 · 无预占人天

当日**未预占 / 预占失败 / 无分配额度**时：

1. 权益准入失败，引导**在线支付**；
2. **默认推荐 1 天或单次 SKU**（**已确认 Q1**），不默认推包月；
3. 支付成功生成 `PackageOrder`，`source=channel_retail_fallback`；
4. 平台 **C 端 1%**（支付成功时），与 personal 相同；
5. **不参与合伙人分润**（**已确认 Q2**）。

### 3.3 渠道兜底 SKU（运营商强制）

**已确认（Q4）**：面向渠道用户（人天额度池兜底），运营商在定价管理中**必须**维护且保持上架：

| 必选 SKU | 用途 |
|----------|------|
| **1 天套餐** | 渠道骑手无预占时的默认兜底之一 |
| **单次套餐** | 渠道骑手无预占时的默认兜底之一 |

- 至少各 1 个**运营商级**生效价格（可按城市/站点覆盖）；
- 下架或缺失时，系统**阻止**该运营商渠道成员进入「自费兜底」路径，并提示运营商补配置；
- 与人天批发价独立；兜底零售价为 C 端在线支付价。

## 4. 平台 1% 技术服务费

| 场景 | 触发 | 对象 |
|------|------|------|
| 个人 / 渠道自费 C 端支付 | **支付成功** | 从运营商收款分账 1% |
| 渠道人天池确认消耗 | **确认消耗** | 额度售卖方 U（B 端规则，见 PRD §5.5） |

渠道自费兜底**仅走 C 端 1%**，不重复计 B 端消耗计提。

## 5. 中途退款（自动）

### 5.1 准入（系统自动校验）

| # | 条件 | 不满足 |
|---|------|--------|
| 1 | 状态 = **服务中** | 拒绝 |
| 2 | **未持有电池** | 「请先归还电池」 |
| 3 | **剩余服务 ≥ 1 单位** | 「剩余不足 1 天/1 次，不可退款」 |
| 4 | `source ≠ day_pool` | 渠道池订单不退骑手（离职回池） |

**剩余单位**：

| SKU | 计算 |
|-----|------|
| 包月 / 7天 / 1天 | 剩余有效时长 ≥ 24h 视为 ≥ 1 天；**不足 24h 不可退** |
| 单次 | 未换电（剩余 1 次）可退；已换电不可退 |

> 1 天 / 单次 SKU：若剩余有效时长 **< 24h 且未使用**，仍视为不足 1 个完整服务单位时按「剩余 ≥ 1 单位」规则——**未换电且距失效 ≥ 1h 可配置**；首版简化为：**单次未用可全额退；1 天未换电且购买后未过 24h 可退**。

### 5.2 应退金额

**天数型（包月 / 7天 / 1天）**：

```
R = round( P × remaining_days / total_days , 2 )
```

**1 天（24h）**：未换电 → `R = P`；已换电 → `R = 0`。

**单次（24h）**：未换电 → `R = P`；已换电 → `R = 0`。

### 5.3 自动退款流程

符合条件 → **无需人工审核** → 终止 `PackageOrder` → 原路退 `R` → 应分/SplitLine 冲正。

### 5.4 退款分佣冲正

| 部分 | 处理 |
|------|------|
| 未确认预收 | 直接冲减，无 SplitLine 冲正 |
| 已确认部分 | 按**原 SplitLine 快照**等比例负向冲正 |

```
平台冲回 = round(原平台份额 × R_confirmed / P_confirmed, 3)
合伙人冲回 = round(原合伙人份额 × R_confirmed / P_confirmed, 3)  // 无合伙人=0
运营商冲回 = 余额（吸收尾差）
```

**渠道自费兜底订单**：仅平台 1% + 运营商，**无合伙人冲正行**。

支付通道：已分账 1% 走**分账回退**；未确认部分仅退预收。

## 6. 实体字段（开发摘要）

| 实体 | 字段 |
|------|------|
| `PkgSku` | `type`（monthly/weekly/daily/single）, `validity_hours`（1天/单次=24）, `channel_fallback_eligible` |

| `RefundRequest` | `auto_approved`, `remaining_units`, `R`, `split_reversal[]` |

## 7. 商务确认清单

| # | 议题 | 确认结果 |
|---|------|----------|
| Q1 | 渠道无预占默认推什么 | **1 天或单次** |
